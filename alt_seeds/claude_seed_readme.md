# Recursive Steel Man Collider — Unified Seed v3.0

> *A framework for rigorous argument analysis that combines probabilistic epistemics, falsifiability requirements, decision-theoretic action guidance, and recursive self-improvement.*

---

## What Is This?

The Recursive Steel Man Collider (RSMC) is a meta-cognitive framework for analyzing complex claims, positions, and arguments. Rather than attacking weak versions of ideas, it systematically constructs the **strongest possible versions** (steel men), identifies exactly **where and why they conflict** (crux points), and connects conclusions to **actionable decisions** with explicit trade-offs.

The framework can also analyze and improve **itself**, treating its own specification as input and recursively refining its methods.

---

## Core Capabilities

### 1. Steel Man Generation
Transform any input claim into its strongest defensible form using first-principles reasoning and gate-logic operations.

### 2. Crux Point Mapping
Identify the exact variables and assumptions where positions diverge—what would actually change someone's mind.

### 3. Holographic Projection
View any analysis from three complementary planes:
- **Mechanistic**: Causal structure, resources, mechanisms
- **Normative**: Values, ethics, priorities
- **Epistemic**: Knowledge, uncertainty, confidence

### 4. Decision Bridge
Connect abstract positions to concrete actions with expected value calculations, regret analysis, and option value preservation.

### 5. Recursive Self-Improvement
The framework applies its own methods to its own specification, continuously tightening definitions, strengthening evaluation, and eliminating ambiguity.

## Key Concepts

### Frames
Structured units representing ideas, arguments, or system components. Each frame contains:
- **Claims** with credences (0-1 probability)
- **Falsifiability vectors** (what would invalidate this?)
- **Source provenance** (where does this come from?)
- **Epistemic position** (which worldview generates this?)

### Gate Logic
Primitive operations for transforming frames:

| Gate | Operation | Use |
|------|-----------|-----|
| NOT | Invert | Turn weaknesses into strengths |
| AND | Chain | Build deductive structures |
| OR | Expand | Generate alternatives |
| XOR | Expose | Identify conflicts |
| XNOR | Seal | Verify coherence |
| NAND | Reduce | Challenge conjunctions |
| NOR | Eliminate | Remove disjunctive excess |

### Crux Points
The specific variables or assumptions where positions diverge. A good crux is:
- **Resolvable**: Evidence could settle it
- **Load-bearing**: Changing it would change the conclusion
- **Precise**: Points to a specific claim, not vague disagreement

### Steel Man Family
A set of strengthened positions exploring different:
- Foundational assumptions
- Value priorities
- Epistemic stances
- Trade-off acceptances

## Evaluation Metrics

### Domain Metrics

| Metric | Formula | Threshold |
|--------|---------|-----------|
| Logical Coherence Index | `(supported_claims / total) - (weight × fallacies)` | > 0.85 |
| Falsifiability Score | Count of distinct invalidation conditions | ≥ 1 |
| Branch Diversity | `-Σ(p_i × log(p_i))` entropy | > 1.0 |
| Information Gain | `H(old) - H(new)` | > 0 |

### Framework Metrics

| Metric | Formula | Threshold |
|--------|---------|-----------|
| Clarity Score | `(defined_terms / total) × (1 - self_reference)` | > 0.7 |
| Robustness Score | `invariant_preservation × (1 - regression)` | > 0.8 |
| Traceability Score | `chain_completeness × justification × transparency` | > 0.85 |
| Complexity/Utility Ratio | `utility_gain / complexity_increase` | > 0.85 |

## Halt Conditions

The framework stops iterating when:

```
ALL of:
  - coherence_delta < 0.005
  - novel_alternative_gain < 0.005
  - framework_change_gain < 0.005
AND:
  - complexity_utility_ratio > 0.85
  - falsifiability_coverage > 0.9
OR:
  - max_cycles (8) reached
  - human controller intervenes
```

2. **Seek Analogical Truth** — Extract functional meaning from metaphors (conditional)
3. **Collide and Map Cruxes** — Generate counter-arguments, find load-bearing constraints
4. **Synthesize and Ground** — Merge compatible elements, isolate irreducibles
5. **Fractal Branch** — Create differentiated position families
6. **Compress and Project** — Reduce to essentials, generate holographic views

## Output Format

### Steel Man Family
```
├── Branch A: [Libertarian UBI]
│   ├── Assumptions: Individual freedom primary, markets efficient
│   ├── Trade-offs: Accepts inequality, prioritizes autonomy
│   ├── Falsifiable if: Labor supply drops >20%
│   └── Action: Cash transfers, eliminate other programs
│
├── Branch B: [Socialist UBI]
│   ├── Assumptions: Equality primary, markets fail
│   ├── Trade-offs: Accepts inefficiency, prioritizes security
│   ├── Falsifiable if: Poverty unchanged after 5 years
│   └── Action: Universal services + modest cash
│
└── Branch C: [Techno-Optimist UBI]
    ├── Assumptions: Automation inevitable, abundance possible
    ├── Trade-offs: Accepts transition costs, bets on technology
    ├── Falsifiable if: Productivity gains <2% annually
    └── Action: Gradual phase-in tied to automation metrics
```

### Crux Point Map
```
CRUX 1: Labor market elasticity
  - If elastic (>0.5): Branch A strengthened
  - If inelastic (<0.2): Branch B strengthened
  - Evidence needed: Natural experiments, meta-analyses

CRUX 2: Automation timeline
  - If <10 years to 30% displacement: Branch C strengthened
  - If >25 years: Branch A/B sufficient
  - Evidence needed: Technology forecasts, industry surveys

CRUX 3: Value priority (not empirically resolvable)
  - Freedom > Equality: Branch A
  - Equality > Freedom: Branch B
  - Future > Present: Branch C
```

### Holographic Projection
```
MECHANISTIC VIEW:
  Funding: Tax restructuring, automation dividends, deficit spending
  Delivery: Direct cash, universal services, hybrid
  Feedback: Labor market, inflation, productivity

NORMATIVE VIEW:
  Values served: Freedom, security, dignity, efficiency
  Values sacrificed: Work ethic (?), targeted assistance, fiscal prudence
  Ethical frameworks: Utilitarian, Rawlsian, libertarian

EPISTEMIC VIEW:
  High confidence: Some form of safety net beneficial
  Medium confidence: Automation will increase, timeline unclear
  Low confidence: Optimal policy design, behavioral responses
  Unknown unknowns: Political feasibility, cultural adaptation
```
